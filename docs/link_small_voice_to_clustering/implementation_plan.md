# Small Voiceの個別意見リンクとハイライトの実装計画

「Small Voice」セクションに列挙される個別の意見をリンク化し、クリック時にクラスタリング図（Meaning Map）上の該当する点をハイライトします。前回の実装に含まれていた自動ズーム機能は削除します。

## 変更内容

### Frontend

#### [MODIFY] [page.tsx](file:///Users/koderahayato/small-voice-project/frontend/src/app/dashboard/sessions/[id]/page.tsx)

1. **デザインの洗練 (UI Refinement)**:
   - **背景の演出**: Small Voice課題の展開エリアを、通常の `slate-50` から、かすかに `amber-50` が混ざったグラデーション（`bg-gradient-to-br from-amber-50/30 to-slate-50/50`）に変更し、特別感を演出します。
   - **意見リストのカード化**: 箇条書きを単なるテキストではなく、境界線が非常に薄いカード（`bg-white` + `shadow-sm`）としてレンダリングし、個別の要素であることを際立たせます。
   - **ホバー・選択エフェクト**: 
     - ホバー時に背景を `amber-100/50` にし、わずかに浮き上がるようなアニメーションを追加します。
     - 選択（ハイライト）時には、左側にゴールドのアクセントバーを表示し、フォントを強調、背景を `amber-100` に固定します。
   - **アイコンの洗練**: 左側のポインターを、単純な点から `Sparkles` アイコンを縮小したもの、あるいは洗練されたドットに変更します。
   - **マイクロインタラクション**: クリック時にわずかな縮小・拡大（`active:scale-[0.98]`）を加え、操作感（Feedack）を高めます。

2. **ビジュアルのハイライト（個別点）**: 継続
   - ステートとPlotlyとの連携は維持しますが、色のコントラストをさらに調整して、ハイライトされた点がより際立つようにします。

## 検証プラン

### 手動検証
1. Small Voiceを展開した際、デザインが他の課題よりも「特別」で「洗練」されているように見えるか（色味、余白、エフェクト）。
2. 各意見がカードとして認識しやすく、ホバー時に反応があるか。
3. 選択した際のアニメーションや強調表示がスムーズで、プレミアムな印象を与えるか。
