# ユーザー登録時の重複エラー修正の確認

## 実施した変更
`frontend/src/components/admin/UserManager.tsx` を修正し、ユーザー作成・編集時のボタン連打防止機能を追加しました。

### 主な変更点
1. **`isSubmitting` ステートの追加**:
   - APIリクエスト中は `true` になり、完了（成功・失敗問わず）後に `false` に戻ります。

2. **ボタンの無効化処理**:
   - 「作成」「保存」「キャンセル」ボタンに `disabled={isSubmitting}` プロパティを追加しました。
   - 送信中はボタンが半透明（opacity-50）になり、クリックできなくなります。

3. **ローディング表示**:
   - 送信中、「作成」ボタンのテキストが「送信中...」に変わり、スピナーが表示されるようになりました。

## 確認方法

1. **管理画面へアクセス**:
   - 管理者権限でログインし、「システム管理」ページへ移動します。
   - 「ユーザー管理」タブを選択します。

2. **新規ユーザー作成のテスト**:
   - 「新規作成」ボタンをクリックします。
   - フォームにメールアドレスと表示名を入力します。
   - 「作成」ボタンを1回クリックします。
   - **確認事項**:
     - ボタンが即座に無効化（グレーアウト）されるか。
     - 「送信中...」と表示されるか。
     - 処理完了までボタンが押せない状態が維持されるか。
   - 処理完了後、再度ボタンがクリック可能（あるいはウィンドウが閉じる）になるかを確認します。

3. **ユーザー編集のテスト**:
   - 既存ユーザーの「編集」ボタンをクリックします。
   - 情報を変更し、「保存」ボタン（チェックマーク）をクリックします。
   - **確認事項**:
     - 保存ボタンとキャンセルボタンが無効化されるか。

この修正により、意図しない重複リクエストによる `Email already exists` エラーの発生を防ぐことができます。
