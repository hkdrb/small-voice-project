# タスク：プロフィール設定更新機能の修正

## 概要
ユーザーがプロフィール（表示名、パスワード）を更新しようとした際、フロントエンドとバックエンドの不整合や、パスワード変更時のセキュリティ要件（現在のパスワード確認）が不足していたため、これらを修正します。

## 課題
- フロントエンドで `name` というフィールドを使っていたが、バックエンドは `username` を期待していた。
- パスワード変更時にバックエンドで必須とされている `current_password` がフロントエンドから送信されていなかった。
- サイドバー等の表示で `name` と `username` が混在していた。

## 完了条件
- [x] 表示名の変更がデータベースに正しく反映される。
- [x] パスワード変更時に現在のパスワードを入力・検証できる。
- [x] 画面上の表示（サイドバー等）が正しく更新される。
