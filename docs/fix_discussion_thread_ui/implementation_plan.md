# 議論スレッドUI改善の実装計画

議論スレッド（右側パネル）において、タイトルが長い場合に閉じるボタンが押し出されたり重なったりする問題を修正し、閉じる際のアニメーションを削除します。

## ユーザーレビューが必要な項目
- アニメーションの削除について、現在は開く際も閉じる際も `duration-100` のトランジションが設定されています。閉じる際のみアニメーションをなくす場合、開く際のアニメーションも合わせて削除するか、あるいは閉じる時だけ即座に消えるように調整します。今回は一旦どちらも削除（または閉じる時だけ削除）の方向で検討します。

## 変更内容

### Frontend

#### [MODIFY] [page.tsx](file:///Users/koderahayato/small-voice-project/frontend/src/app/dashboard/sessions/[id]/page.tsx)

- **閉じるボタンのスタイル修正**:
  - タイトルを表示する `div` に `min-w-0` と `flex-1` を追加。
  - `h3` （タイトル）には既に `line-clamp-1` が適用されているため、これでタイトルが領域を超えた際に適切に省略され、ボタンが押し出されるのを防ぎます。
  - 閉じるボタンに `shrink-0` を追加し、ボタンのサイズが維持されるようにします。

- **アニメーションの削除**:
  - 右側パネルのコンテナから `transition-all duration-100` クラスを削除します。
  - 閉じる動作を即座に反映させるため、`activeIssue` が `null` になる際の幅の変化をアニメーションなしにします。

## 検証計画

### 自動テスト
- 特になし（UIスタイルの修正であるため）

### 手動検証
1. 非常に長いタイトルの課題（Issue）を選択し、右側パネルが開いた際に閉じるボタンが正常な位置（右端）に表示されているか確認する。
2. 閉じるボタンを押した際、パネルがアニメーションなしで即座に閉じることを確認する。
3. パネルが閉じる際、左側のグラフ（Plotly）のサイズが即座に追従して再描画されるか確認する。
