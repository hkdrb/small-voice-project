# Small Voice システム利用マニュアル

## 1. はじめに
本マニュアルは、組織内ブロードリスニングシステム「Small Voice」の操作方法を解説するものです。
本システムは、日々の業務における気づきや改善案、システムに関するフィードバックなどの「小さな声」を収集し、AI分析を通じて組織改善に活かすことを目的としています。

利用可能な機能は、ユーザーの権限（ロール）によって異なります。

---

## 2. 共通機能 (全ユーザー)
すべてのユーザーが利用可能な基本機能です。

### 2.1 ログイン
1.  **アカウント取得**: アカウントは**システム管理者**によって作成・招待されます。招待メールのリンクまたは配布された情報を使用します。
2.  **ログイン**: ログイン画面にアクセスし、情報を入力してログインします。

### 2.2 ダッシュボードの閲覧
ログイン直後に表示される画面です。
*   **分析レポート**: 自身の組織で公開された分析結果（セッション）の一覧が表示されます。
    *   ※一般ユーザーには「公開中」のレポートのみが表示されます。

### 2.3 解決策チャットとAI分析 (公開レポートのみ)
公開された分析レポートに対して、議論に参加したり、その議論のAI分析結果を閲覧できます。
1.  **レポート閲覧**: ダッシュボードの「レポート閲覧」タブから、閲覧したい分析セッションをクリックします。
2.  **トピックマップ**: アンケート回答の分布マップを確認します。
3.  **課題リスト**: データから抽出された重要な課題一覧を確認します。
4.  **みんなの提案分析**: チャットでの議論内容をAIが分析した結果（トレンド、注目アイデア）を閲覧できます（※管理者が公開している場合）。
5.  **チャット参加**: 画面下部の「みんなの提案チャット」にて、意見投稿、リプライ、いいねが可能です。
    *   自身のコメントは「編集」ボタン (鉛筆アイコン) から修正できます。

### 2.4 通知機能
システム内で重要なイベントが発生した際、ダッシュボード右上のヘッダーにある**ベルアイコン**に通知が届きます。

#### 基本ルール
*   **自分除外**: 自分自身の操作（コメント投稿、レポート公開など）による通知は、自身には届きません。
*   **日本時間**: 通知時刻はすべて**日本時間 (JST)** で表示されます。
*   **表示件数**: 通知リストには最新**99件**まで表示されます。
*   **自動遷移**: レポート(課題)関連の通知をクリックすると、ページ遷移と同時に**該当する議論スレッドが自動的に開きます**。

#### 通知の種類と対象者

| カテゴリ | 内容 | 公開状態による制限 | 対象者 |
| :--- | :--- | :--- | :--- |
| **全体のお知らせ** | アンケート公開、レポート公開、AI分析の更新、雑談のAI提案公開 | **公開済み**のもの限定 | 組織の全メンバー |
| **みんなの雑談** | 雑談掲示板への新規投稿・返信 | 常に通知 | 組織の全メンバー |
| **レポート議論** | 課題ごとのチャットへの新着投稿、AI分析の更新 | レポートが**公開中**なら全員、**未公開**なら管理者のみ | 上記条件による |
| **申請のやり取り** | フォーム申請に対する承認・却下・コメント返信 | 常に通知 | 申請した本人のみ |
| **管理業務** | 一般ユーザーからの新規フォーム申請・再申請・コメント投稿 | 常に通知 | 全ての組織管理者・システム管理者 |

#### 通知の使い方
1.  **バッジ**: 未読の通知がある場合、ベルアイコンに件数（最大 99+）が表示されます。
2.  **確認と遷移**: アイコンをクリックすると通知リストが表示されます。通知をクリックすると該当ページへ直接遷移し、自動的に既読となります。
3.  **すべて既読**: 通知リスト内の「全て既読にする」ボタンで、一括して既読状態にできます。

## 3. 一般ユーザー
組織の一般メンバー向けの機能です。

### 3.1 アンケートへの回答
1.  ダッシュボードの「アンケート回答」タブ（または回答受付中エリア）を確認します。
2.  回答したいアンケートを選んで「回答する」ボタンを押し、設問に回答して送信します。

### 3.2 アンケートフォームの提案 (ボトムアップ申請)
現場の気づきを元に、新しいアンケート実施を管理者に提案できます。

1.  ダッシュボードの「フォーム申請」タブを選択し、「作成申請」ボタンをクリックします。
2.  タイトル、説明、独自の設問を作成します。
3.  **「申請」ボタン**を押します。
    *   ステータスは「申請中」となり、管理者が確認します。
    *   管理者が承認すると「承認済み（停止中）」となり、公開されると回答が可能になります。
    *   却下された場合は、修正・再申請ができます。

## 4. 組織管理者
各組織（部署・チーム）の管理権限を持つユーザー向けの機能です。

### 4.1 フォーム作成・管理 (承認フロー)
一般ユーザーからの申請を審査し、公開コントロールを行います。

1.  **審査**:
    *   ダッシュボードの「フォーム作成・管理」タブで「申請中」のフォームを確認します。
    *   内容を確認し、**承認 (`Check`)** または **却下 (`X`)** を選択します。
2.  **結果のダウンロード**:
    *   アンケート回答結果は、ダウンロードボタンから**CSV形式**で全回答データをエクスポートできます。
3.  **公開/停止**:
    *   承認済み（停止中）のフォームに対し、**公開 (`Eye`)** アイコンを押すと、組織内の全メンバーに公開されます。
    *   回答受付を終了する場合は、再度アイコンを押して「非公開」にします。
3.  **新規作成・編集**:
    *   管理者が自ら「新規作成」を行うことも可能です。また、不適切な申請を編集・削除することもできます。

### 4.2 AI分析の実行
蓄積されたデータを分析し、レポートを作成します。

1.  ダッシュボードの「データ分析実行」タブを選択します。
2.  **分析対象の選択**: 分析したい「フォーム」と、その中の特定の「質問」を選択します。
3.  **実行**: レポート名（任意）を入力し、「分析を開始する」をクリックします。
    *   分析には数分かかる場合があります。
4.  **公開**: 分析完了後、レポート内容を確認し、問題なければ**「公開する」**ボタンで一般メンバーに公開します。

### 4.3 CSVデータのインポート
外部システムや過去のアンケートデータを分析用に取り込むことができます。

1.  ダッシュボードの「CSVインポート」タブを選択します。
2.  ファイル選択エリアをクリックし、CSVファイルをアップロードします。
3.  フォームタイトルを入力し、分析対象とする列（カラム）を選択します。
    *   1行目を「質問文」、2行目以降を「回答」として認識します。
4.  **「インポートを実行」**をクリックすると、データがフォームとしてシステムに登録され、分析が可能になります。

### 4.4 みんなの提案分析 (チャット分析)
解決策チャットで蓄積された議論に対しても、AI分析を実行できます。

1.  対象の分析レポート詳細画面を開きます。
2.  画面下部の「みんなの提案分析」セクションにある**「分析を実行」**（または「再分析」）ボタンを押します。
    *   チャット上のコメントをAIが分析し、トレンドや注目アイデアを抽出します。
3.  分析結果を確認し、**「公開」**することで一般メンバーにも表示されます。

### 4.5 メンバーリストの確認
1.  ダッシュボードの「メンバーリスト」タブを選択します。
2.  現在組織に参加しているメンバーの一覧（名前、メール、権限）を確認できます。
    *   ※メンバーの追加・招待（アカウント作成）は、**システム管理者**のみが実行可能です。

## 5. システム管理者
システム全体の保守・管理を行う特権ユーザー向けの機能です。

### 5.1 組織の作成
*   新しい利用組織（テナント）を作成し、初期の組織管理者を割り当てます。
*   ダッシュボードのシステム管理エリア（`/admin/system`）から実行します（※一部機能は開発用DB操作が必要な場合があります）。

### 5.2 ユーザー管理
*   **アカウント作成・招待**: 新しいユーザーアカウントを作成し、招待メールまたはパスワード設定リンクを発行します。
*   組織へのメンバー割り当てを行います。

### 5.3 全機能へのアクセス
*   システム管理者は、すべての組織のデータおよび機能に対して、組織管理者と同等以上のアクセス権限を持ちます。
*   システム全体に関わる設定やメンテナンスを行います。
