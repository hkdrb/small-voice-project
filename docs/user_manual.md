# Small Voice システム利用マニュアル

## 1. はじめに
本マニュアルは、組織内ブロードリスニングシステム「Small Voice」の操作方法を解説するものです。
本システムは、日々の業務における気づきや改善案、システムに関するフィードバックなどの「小さな声」を収集し、AI分析を通じて組織改善に活かすことを目的としています。

利用可能な機能は、ユーザーの権限（ロール）によって異なります。

---

## 2. 共通機能 (全ユーザー)
すべてのユーザーが利用可能な基本機能です。

### 2.1 ログイン
1.  **アカウント取得**: アカウントは**システム管理者**によって作成・招待されます。招待メールのリンクまたは配布された情報を使用します。
2.  **ログイン**: ログイン画面にアクセスし、情報を入力してログインします。

### 2.2 ダッシュボードの閲覧
ログイン直後に表示される画面です。サイドバー（モバイルの場合はメニュー）から各機能にアクセスできます。

*   **アンケート回答**: 回答受付中のアンケート一覧が表示されます。
*   **レポート閲覧**: 公開された分析レポート（課題解決セッション）の一覧が表示されます。
*   **雑談掲示板**: 組織内の「みんなの声」を共有する掲示板です。

### 2.3 雑談掲示板 (みんなの声)
日々の業務での気づきや、ふとした思いつきを気軽に投稿できる場所です。
*   **投稿**: 画面右下の「＋」ボタン（または投稿ボタン）から新規投稿ができます。
*   **返信・いいね**: 他のメンバーの投稿に対して、返信や「いいね」でリアクションできます。
*   **AI分析**: 管理者はここでの会話データをAI分析し、新しいアンケートの種などを発見します。

### 2.4 解決策チャットとAI分析 (公開レポートのみ)
公開された分析レポートに対して、議論に参加したり、その議論のAI分析結果を閲覧できます。
1.  **レポート閲覧**: サイドバーの「レポート閲覧」から、閲覧したい分析セッションをクリックします。
2.  **トピックマップ**: アンケート回答の分布マップを確認します。
3.  **課題リスト**: データから抽出された重要な課題一覧を確認します。
4.  **みんなの提案分析**: チャットでの議論内容をAIが分析した結果（トレンド、注目アイデア）を閲覧できます（※管理者が公開している場合）。
5.  **チャット参加**: 画面下部の「みんなの提案チャット」にて、意見投稿、リプライ、いいねが可能です。
    *   自身のコメントは「編集」ボタン (鉛筆アイコン) から修正できます。

### 2.5 通知機能
システム内で重要なイベントが発生した際、ダッシュボード右上のヘッダーにある**ベルアイコン**に通知が届きます。

#### 基本ルール
*   **自分除外**: 自分自身の操作（コメント投稿、レポート公開など）による通知は、自身には届きません。
*   **自動遷移**: レポート(課題)関連の通知をクリックすると、ページ遷移と同時に**該当する議論スレッドが自動的に開きます**。

#### 通知の種類と対象者

| カテゴリ | 内容 | 公開状態による制限 | 対象者 |
| :--- | :--- | :--- | :--- |
| **全体のお知らせ** | アンケート公開、レポート公開、AI分析の更新 | **公開済み**のもの限定 | 組織の全メンバー |
| **みんなの雑談** | 雑談掲示板への新規投稿・返信 | 常に通知 | 組織の全メンバー |
| **雑談の分析** | 雑談データのAI分析による提案作成 | 管理者のみ（公開後は全員） | 管理者（公開時は全員） |
| **レポート議論** | 課題ごとのチャットへの新着投稿、AI分析の更新 | レポートが**公開中**なら全員、**未公開**なら管理者のみ | 上記条件による |
| **申請のやり取り** | フォーム申請に対する承認・却下・コメント返信 | 常に通知 | 申請した本人のみ |
| **管理業務** | 一般ユーザーからの新規フォーム申請・再申請・コメント投稿 | 常に通知 | 全ての組織管理者・システム管理者 |

---

### 2.6 プロフィール設定
サイドバー最下部の「プロフィール設定」ボタンから、以下の情報を変更できます。
*   **名前**: システム内での表示名。
*   **パスワード**: ログインパスワード（変更する場合のみ入力）。

## 3. 一般ユーザー
組織の一般メンバー向けの機能です。

### 3.1 アンケートへの回答
1.  サイドバーの「アンケート回答」タブを確認します。
2.  回答したいアンケートを選んで「回答する」ボタンを押し、設問に回答して送信します。
3.  回答内容は匿名化されて分析に使用されます。

### 3.2 フォーム申請 (ボトムアップ申請)
現場の気づきを元に、新しいアンケート実施を管理者に提案できます。

1.  サイドバーの「フォーム申請」タブを選択し、「フォーム申請作成」ボタンをクリックします。
2.  タイトル、説明、独自の設問を作成し、**「申請」ボタン**を押します。
3.  申請後、**チャット画面（編集画面）に遷移**します。ここで管理者と申請内容についてメッセージのやり取りが可能です。
4.  **ステータスの変化**:
    *   **申請中**: 管理者が確認中の状態です。
    *   **承認済み**: 管理者が承認した状態です。その後、管理者が「公開」操作を行うと、全メンバーが回答可能になります。
    *   **却下**: 修正が必要な場合など。修正して再申請が可能です。
    *   **承認済み（停止中）**: 管理者側でのステータス表記です。

## 4. 組織管理者
各組織（部署・チーム）の管理権限を持つユーザー向けの機能です。
※一般ユーザーの機能（アンケート回答、雑談投稿など）も全て利用可能です。

### 4.1 フォーム作成・管理
アンケートフォームの作成、および一般ユーザーからの申請の管理を行います。

1.  **一覧確認**: サイドバーの「フォーム作成・管理」タブでフォーム一覧を確認します。
2.  **承認フロー**:
    *   「申請中」のフォームを確認し、**承認 (`Check`)** または **却下 (`X`)** を選択します。
    *   承認されたフォームは**「停止中」**となり、公開の準備が整った状態になります。
3.  **チャット機能**:
    *   申請中のフォーム詳細画面から、申請者とチャットで連絡を取ることができます（設問の修正依頼など）。
4.  **公開/停止**:
    *   停止中のフォームに対し、**公開 (`Eye`)** アイコンを押すと「公開中」となり、組織内の全メンバーが回答可能になります。
    *   回答受付を終了する場合は、再度アイコンを押して「非公開」にします。
5.  **結果のダウンロード**:
    *   各フォームの「CSVダウンロード」ボタンから、全回答データをエクスポートできます。

### 4.2 AI分析の実行 (アンケート分析)
蓄積されたアンケート回答データを分析し、レポートを作成します。

1.  サイドバーの「データ分析実行」タブを選択します。
2.  **分析対象の選択**: 分析したい「フォーム」と、その中の特定の「質問」を選択します。
3.  **実行**: レポート名（任意）を入力し、「分析を開始する」をクリックします。
4.  **公開**: 分析完了後、レポート内容を確認し、問題なければ**「公開する」**ボタンで一般メンバーに公開します。

### 4.3 雑談のAI分析 (作成フォームの提案)
雑談掲示板の会話データをAIが分析し、組織の関心事や課題を抽出して、新しいアンケート（フォーム）の作成を提案します。

1.  「雑談掲示板」画面を開きます。
2.  （管理者のみ）画面右側に「作成フォームの提案」エリアが表示されます。
3.  **分析実行**: 対象期間を指定して「AI分析を実行」を行います。
4.  **提案の活用**: 分析結果として推奨されるアンケートテーマが表示されます。「この提案でフォームを作成」ボタンを押すと、提案内容に基づいたフォームの下書きが自動作成されます。

### 4.4 みんなの提案分析 (解決策チャット分析)
解決策チャット（レポート内での議論）に対しても、AI分析を実行できます。

1.  対象の分析レポート詳細画面を開きます。
2.  画面下部の「みんなの提案分析」セクションにある**「分析を実行」**（または「再分析」）ボタンを押します。
3.  分析結果を確認し、**「公開」**することで一般メンバーにも表示されます。

### 4.5 CSVデータのインポート
外部システムや過去のアンケートデータを分析用に取り込むことができます。
サイドバーの「CSVインポート」タブから実行します。1行目を質問文、2行目以降を回答データとして読み込みます。

### 4.6 メンバーリストの確認
1.  サイドバーの「メンバーリスト」タブを選択します。
2.  現在組織に参加しているメンバーの一覧を確認できます。
3.  表示項目: ユーザー名、メールアドレス、システム権限、所属組織と組織内権限。
    *   ※メンバーの追加・招待（アカウント作成）は、**システム管理者**のみが実行可能です。

## 5. システム管理者
システム全体の保守・管理を行う特権ユーザー向けの機能です。
サイドバー下部の「システム管理」リンク（または `/admin/system`）から専用ダッシュボードにアクセスします。

### 5.1 組織管理
「組織管理」タブで以下の操作を行います。
*   **組織一覧**: システムに登録されている全組織を確認できます。
*   **新規作成**: 新しい組織（部署・プロジェクト）を作成します。
*   **編集・削除**: 既存組織の名称変更や削除を行います。

### 5.2 ユーザー管理
「ユーザー管理」タブで以下の操作を行います。
*   **ユーザー一覧**: 全登録ユーザーを確認できます。
*   **新規作成 (招待)**: アカウントを作成し、招待メールを送信します。
*   **権限設定**:
    *   **システム権限**: システム管理者か否かを設定します。
    *   **組織への所属**: 各組織への所属有無と、組織内でのロール（一般/管理者）を設定します。
*   **編集・削除**: 既存ユーザー情報の変更やアカウント削除を行います。
