# タスク: valuesデータにおけるクラスタリングの改善

## 背景
values（価値観）カテゴリのテストデータにおいて、クラスタリング結果がすべて一つのカテゴリに集約されてしまう事象が発生している。これはデータの重複度が高く、密度ベースのアルゴリズム（HDBSCAN）が適切に分離できていないことが原因と考えられる。

- [x] `backend/services/analysis.py` のクラスタリングロジックの修正
    - [x] HDBSCANでクラスタが検出されなかった場合のKMeansフォールバック処理を追加
- [x] `scripts/generate_test_data.py` のvaluesデータ生成ロジックの改善
    - [x] テンプレートの拡充と多様性の向上
- [x] 修正後のテストデータ再生成と動作確認
